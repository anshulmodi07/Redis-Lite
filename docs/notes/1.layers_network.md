# Comprehensive Notes on Networking Basics

The foundation of modern networking relies on dividing complex communication tasks into distinct layers. This structure is universally explained using the **OSI (Open Systems Interconnection) Model**, which contains 7 layers. 

Every layer serves the layer above it and relies on the layer below it. When data travels from your device to a server, it moves from **Layer 7 down to Layer 1**, crosses the physical medium, and moves from **Layer 1 up to Layer 7** on the receiving machine.

---

## The 7 Layers of the OSI Model & Famous Protocols



### 1. The Physical Layer (Layer 1)
The Physical layer deals with the hardware, mechanical, and electrical specifications of the network. It handles the transmission of raw, unstructured bits (`0`s and `1`s) over a physical medium. It defines voltage levels, pin configurations, cable types, and radio frequencies.

* **Ethernet (Physical specs like 100BASE-T / 10GBASE-T):** Defines cable specifications (like Cat6), signal frequencies, and connector shapes (like RJ45) for wired connections.
* **DSL (Digital Subscriber Line):** Uses standard copper telephone lines to transmit high-frequency digital signals alongside traditional analog voice data.
* **Wi-Fi (Physical specs like 802.11ax/be):** Dictates radio frequency bands (2.4GHz, 5GHz, 6GHz), signal modulation, and antenna requirements for wireless transmission.

---

### 2. The Data Link Layer (Layer 2)
The Data Link layer sets up a reliable link across a physical connection. It packages raw bits from Layer 1 into discrete units called **Frames**. It handles physical addressing via **MAC (Media Access Control) addresses** to move data between two nodes sharing the *same local network segment*. It also manages error detection (using cyclic redundancy checks).

* **Ethernet (IEEE 802.3 MAC/LLC):** Standardizes how local devices coordinate access to a shared physical medium to prevent data collisions.
* **ARP (Address Resolution Protocol):** The bridge between Layer 3 and Layer 2. When a device knows a target's IP address but needs its physical MAC address to send a frame inside the local network, it broadcasts an ARP request.
* **PPP (Point-to-Point Protocol):** A direct communication protocol established directly between two network nodes, without routers or switches in between (commonly used by Internet Service Providers to connect residential modems to their central infrastructure).

---

### 3. The Network Layer (Layer 3)
The Network layer is responsible for routing data across *different* networks from the original source host to the final destination host. It handles logical addressing via **IP addresses** and uses routers to determine the most optimal path for data. Data at this layer is called **Packets**.



* **IP (Internet Protocol - IPv4 / IPv6):** The core protocol of the internet. It assigns unique logical addresses to devices and dictates how packets are structured, addressed, fragmented, and routed.
* **ICMP (Internet Control Message Protocol):** A diagnostics and error-reporting protocol. Network devices use it to send operational messages (e.g., "Destination Unreachable"). Popular utilities like `ping` and `traceroute` rely directly on ICMP.
* **OSPF (Open Shortest Path First):** An internal routing protocol used within an organization's private network. It maps out all available paths and dynamically calculates the mathematically "shortest" (fastest) route for packets.
* **BGP (Border Gateway Protocol):** The routing protocol that binds the global internet together. It manages how massive core networks owned by different Internet Service Providers (ISPs) exchange routing information to deliver packets globally.

---

### 4. The Transport Layer (Layer 4)
The Transport layer manages end-to-end, process-to-process communication. It takes data from the upper layers, slices it into smaller portions, and ensures it reaches the correct application on the target host using **Port Numbers** (e.g., port 80 for HTTP, port 6379 for Redis). Data here is called **Segments** (in TCP) or **Datagrams** (in UDP).

* **TCP (Transmission Control Protocol):** A connection-oriented, highly reliable protocol. It establishes a session via a "3-way handshake" (SYN, SYN-ACK, ACK). It guarantees data delivery, ensures packets are reassembled in the exact order they were sent, handles flow control (preventing a fast sender from crashing a slow receiver), and automatically retransmits lost data.
* **UDP (User Datagram Protocol):** A connectionless, lightweight alternative to TCP. It operates on a "fire-and-forget" basis, sending data immediately without setting up a connection, checking for errors, or tracking order. It sacrifices perfect reliability for raw speed, making it essential for live streaming, online gaming, and VoIP.

---

### 5. The Session Layer (Layer 5)
The Session layer establishes, coordinates, maintains, and terminates conversational "sessions" between applications on separate devices. It manages authentication, controls who speaks when (duplex management), and inserts checkpoints into data transfers so a stream can resume from its last known stable position if a network drop occurs.

* **NetBIOS (Network Basic Input/Output System):** A legacy local network protocol that allows applications on separate computers to locate and establish sessions with each other to share resources like files and printers.
* **RPC (Remote Procedure Call):** A protocol that lets a software program trigger a function or execute a block of code on a remote server as if it were running natively on the local machine.
* **SOCKS (Socket Secure):** A protocol that routes network packets between a client and a server through an intermediary proxy server, evaluating and establishing the session before traffic begins flowing.

---

### 6. The Presentation Layer (Layer 6)
The Presentation layer acts as the network's data translator and syntax checker. It formats, structures, and presents upper-layer data into a standardized format that the receiving machine can interpret. It handles character encoding translations (e.g., converting EBCDIC to ASCII/UTF-8), data compression, and cryptographic encryption/decryption.

* **TLS (Transport Layer Security):** The modern successor to SSL. It provides end-to-end encryption and identity verification for network traffic (crucial for wrapping insecure layer 7 protocols, turning HTTP into HTTPS).
* **MIME (Multipurpose Internet Mail Extensions):** Extends format capabilities for text-based communications. It allows emails to support binary attachments (like PDFs and ZIP files), non-ASCII character sets, and rich HTML formatting.
* **Data Formatting Standards (JSON / XML / ASCII):** While these are markup and encoding structural standards rather than pure operational wire protocols, the Presentation layer is responsible for marshaling/unmarshaling these formats into memory structures applications can consume.

---

### 7. The Application Layer (Layer 7)
The Application layer is closest to the end user. It does not represent the application itself (like Chrome or Outlook), but rather the specialized **high-level protocols** that those applications interact with to deliver meaningful network services (like browsing web pages, sending emails, or transferring files).

* **HTTP / HTTPS (Hypertext Transfer Protocol / Secure):** The foundational communication language of the World Wide Web. It governs how web clients request and how web servers deliver resources like HTML files, images, and API payloads.
* **DNS (Domain Name System):** The distributed directory of the internet. Because computers route data using numeric IP addresses but humans prefer names, DNS translates a human-readable domain (like `example.com`) into its corresponding machine IP address.
* **SMTP (Simple Mail Transfer Protocol):** The standard text-based protocol used to push and route emails outward from an email client to a destination mail server.
* **FTP / SFTP (File Transfer Protocol / Secure):** Protocols designed specifically for uploading and downloading files across remote file systems. SFTP utilizes an underlying SSH connection to keep the file contents and credentials encrypted.
* **SSH (Secure Shell):** Provides a completely encrypted tunnel for remotely logging into servers, managing infrastructure, and securely running administrative commands via a terminal.
